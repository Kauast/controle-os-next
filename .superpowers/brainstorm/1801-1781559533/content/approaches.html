3 abordagens para o Dispatch Board

Cada uma com trade-offs diferentes. Clique para selecionar.

A

CSS Grid nativo + dnd-kit

Recomendada. Layout inteiramente em CSS Grid. Drag-and-drop com dnd-kit (já popular em projetos Next.js). Realtime via Supabase Realtime (já instalado na VPS). Push mobile via endpoint que o app Capacitor consome.

Prós

  • Zero dependências pesadas
  • Supabase Realtime já disponível
  • Performance excelente
  • dnd-kit tem suporte a toque (mobile)

Contras

  • Grid com horários requer math de posicionamento
  • Modal de confirmação precisa ser feito do zero
B

FullCalendar (biblioteca pronta)

Biblioteca de calendário completa com drag-and-drop, visão dia/semana, recursos (equipes). Realtime via polling ou Supabase. Mais rápido de implementar a parte de calendário.

Prós

  • Calendário pronto com D&D embutido
  • Visão dia/semana já implementada
  • Menos código a escrever

Contras

  • +200KB bundle
  • Customização visual limitada (difícil seguir o design system RB)
  • Fila de OSs pendentes precisa ser construída separada
C

CSS Grid + React DnD (mais baixo nível)

Similar à A mas usando react-dnd ao invés de dnd-kit. Mais verboso, mais controle fino sobre colisões e drops customizados.

Prós

  • Controle total do comportamento
  • Bem documentado

Contras

  • Mais código boilerplate que dnd-kit
  • Suporte a toque limitado sem addon
  • dnd-kit é mais moderno e mantido
Recomendação

A opção A (CSS Grid + dnd-kit + Supabase Realtime) é a mais alinhada com o stack atual — sem adicionar peso ao bundle, mantém o design system RB, e aproveita a infra Supabase que já está rodando na VPS.
